The Challenge

Optimizations:

It is clear that in order to make the program faster, you need to find a way to reduce the number of calculations you are going to make. If you use Brute Force there seem to be over 60 milion possibilities to place the blocks on the grid.

First optimization ... Whitespace counting:

One of the interesting observations to be made is that every block you place on the grid will cover 5 cells of the grid. That means that if you count the number of free consecutive blocks, it must be able to be divided by 5. If this is not the case, it will never fit all the blocks. Implementing this can save you a lot of calculations on a grid that will never have a solution.

This is the optimization that Jan Sipke explained on his website and implemented in his code.
